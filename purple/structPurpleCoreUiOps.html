Detailed Description

Callbacks that fire at different points of the initialization and teardown of libpurple, along with a hook to return descriptive information about the UI.

Member Data Documentation

◆ debug_ui_init

void(* PurpleCoreUiOps::debug_ui_init) (void)

Called just after the debug subsystem is initialized, but before just about every other component's initialization. The UI should use this hook to call purple_debug_set_ui_ops() so that debugging information for other components can be logged during their initialization.

◆ get_ui_info

GHashTable *(* PurpleCoreUiOps::get_ui_info) (void)

◆ quit

void(* PurpleCoreUiOps::quit) (void)

◆ ui_init

void(* PurpleCoreUiOps::ui_init) (void)

Called after all of libpurple has been initialized. The UI should use this hook to set all other necessary UiOps structures.
